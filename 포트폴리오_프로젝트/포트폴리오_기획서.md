# 포트폴리오 기획서

레퍼런스 분석(adhamdannaway.com, lynnandtonic.com)과 `colors.css`(theVenti 팔레트)를 바탕으로 작성.

## 1. 나의 포지셔닝
- **목표 직무**: 풀스택 개발자
- **경력 수준**: 신입
- **지원 회사 성격**: 스타트업, 대기업, 에이전시
- **나만의 강점**: 넓은 분야(풀스택) · 소통 능력 · 호기심
- **포지셔닝 한 줄**: "디자인 감각을 갖춘, 호기심 많은 AI 협업형 풀스택 개발자"

## 2. Hero 섹션 컨셉
**참고**: adhamdannaway.com — "designer / &lt;coder&gt;"처럼 헤드라인 하나로 정체성을 압축 전달하는 방식

- **헤드라인 스타일**: 기술중심형 → adham처럼 두 단어의 대비 구조를 차용하되, 코드 구문 자체를 헤드라인 디자인 요소로 사용
  - 예시 문구: `<Developer designer />` 또는 `const developer = curious + communicative;` 형태로 실제 코드 문법을 헤드라인에 녹여 "기술 중심"임을 시각적으로 증명
- **시각적 요소**: 미니멀 + 애니메이션
  - 타이핑 애니메이션(코드가 한 줄씩 입력되는 느낌)으로 헤드라인 등장
  - 배경은 과하지 않게, 커서를 따라 미세하게 움직이는 블롭(blob) 정도만
- **색상 톤**: 파스텔톤 + 보라색
  - `theme.js`에 정의된 `custom.warmLavender (#B39DDB)`를 Hero 전용 파스텔 포인트로 사용하고, 본문 CTA·강조 요소는 `colors.css`의 진한 보라(`--color-btn-purple #7A00A0`)로 대비를 줌 → 전체 사이트 톤(theVenti 팔레트)과 이질감 없이 Hero만 살짝 부드럽게 열어주는 효과
- **차별화 포인트**: **"코드와 일러스트의 만남"**
  - 본인이 직접 그린 손그림 일러스트를 SVG 라인 드로잉으로 변환해, 코드 헤드라인 옆에 배치 → 마우스를 올리면 선이 그려지는(stroke-dashoffset) 애니메이션으로 완성
  - "개발자인데 그림도 그린다"가 아니라 "그림 그리듯 코드를 짠다"는 인상을 Hero에서 바로 각인

## 3. About Me 섹션 구성
**참고**: lynnandtonic.com — "Hey! I'm Lynn..."로 시작하는 1인칭 대화체, 개인적 취향을 스스럼없이 드러내는 톤

- **개발 시작 스토리**: 평소 웹디자인에 관심을 갖고 알아보다가 개발로 자연스럽게 확장된 흐름
  - 문구 예시: "예쁜 화면을 보면 '이건 어떻게 만들었을까' 궁금해하다가, 어느새 직접 만들고 있었어요."
- **핵심 가치관**: 디자인 · 논리적 사고 · 문제 해결 능력
  - "예쁘다"에서 끝나지 않고 "왜 이 구조가 사용자에게 편한가"까지 파고드는 태도로 서술
- **개인적 매력**: 일러스트 제작, 소품 만들기, 호기심, 새로운 기술
  - lynnandtonic처럼 대화체로: "코드가 안 풀리면 잠깐 펜을 들고 그림을 그려요. 손으로 뭔가 만드는 걸 좋아해서 소품도 종종 만듭니다. 새로운 기술은 일단 써보고 판단하는 편이에요."
  - 취미(일러스트/소품)를 Hero의 SVG 드로잉 요소와 연결해 사이트 전체에 일관된 서사로 엮기
- **성장 목표**: AI를 활용한 풀스택 개발자
  - "지금은 AI를 도구로 잘 쓰는 개발자가 되는 게 목표예요. 빠르게 배우고, 빠르게 만들고, 빠르게 검증하는 사이클을 즐깁니다."

## 4. Skill 섹션 차별화
**참고**: adhamdannaway.com — Figma 95%, Design Systems 90% 같은 퍼센티지 스킬바 표현

- **주력 기술(5개)**: HTML, CSS, JavaScript, Java, Python
- **표현 방식**: 프로그래스 바 (색상 그라디언트로 시각화 — 0%는 연보라, 100%는 진보라)
- **숙련도 기준**: HTML 90% / CSS 80% / JavaScript 70% / Java 70% / Python 80%
- **추가 표시 정보**: 신입이라 "경력 연수" 대신
  - 완주한 프로젝트 수 (예: "완성한 프로젝트 6개")
  - 학습 중인 기술 스택 태그 ("Learning" 배지 — React, Supabase 등)
  - AI 협업 경험 표시 (예: "Claude/AI 페어프로그래밍 활용" 배지)

## 5. 전체 디자인 방향
- **전체 컬러 팔레트**: `colors.css` (theVenti) 그대로 사용
  - Primary `#7A00A0` / Secondary `#FF5B28` / Accent `#014924` / bg-cream `#F9F0E1`
  - Hero에서만 pastel(`warmLavender #B39DDB`)로 부드럽게 열고, 아래 섹션으로 내려갈수록 진한 보라·오렌지로 힘을 주는 톤 그라데이션 전략
- **폰트 스타일**: 클래식 (제목만 세리프 포인트 폰트로 대비, 본문은 Roboto 유지)
- **레이아웃 스타일**: 매거진 — 비대칭 그리드, 프로젝트 섹션은 에디토리얼 스타일로 사진·타이틀·짧은 카피를 자유롭게 배치
- **애니메이션 수준**: 화려함 — 스크롤 트리거 애니메이션, Hero 타이핑/드로잉 애니메이션, 프로젝트 카드 호버 네온 효과를 핵심 섹션에 집중 배치

## 6. 차별화 전략
- **경쟁자 대비 내 강점**: 대부분의 신입 개발자 포트폴리오는 "기술 스택 나열형"인데, 나는 디자인 감각(일러스트) × 논리적 개발 능력 × AI 활용 능력이 결합된 지원자임. 넓은 분야를 다루는 풀스택 역량 + 소통 능력은 협업 마찰이 적은 신입이라는 신호로 이어짐.
- **타겟 어필 포인트 (채용 담당자가 매력을 느낄 요소)**:
  1. 비주얼 감각 있는 개발자는 드묾 — 디자이너와 직접 소통하며 UI를 이해하고 구현할 수 있다는 신호 (에이전시/스타트업에 특히 강한 어필)
  2. AI 활용 능력을 실제 프로젝트에 녹여 보여줌 — 현재 채용 시장에서 가장 주목받는 역량이자 빠른 적응력의 증거
  3. 호기심 기반의 학습 태도가 구체적 결과물(완주 프로젝트 수, 학습 태그)로 증명됨
  4. 소통 능력이 About/프로젝트 설명 문체 자체에서 드러남 — 문서화 잘하는 개발자라는 인상
- **독창적 아이디어 (아무도 하지 않은 나만의 접근법)**:
  - **"AI 협업 로그" 섹션 신설**: 실제 프로젝트에서 AI와 주고받은 프롬프트/의사결정 과정 일부를 짧게 공개하는 미니 케이스 스터디 — "어떻게 생각하고 문제를 푸는가"를 채용 담당자에게 직접 보여주는 방식
  - **일러스트 → SVG 인터랙션화**: 손그림을 실제 UI 인터랙션 요소(호버 시 라인 드로잉)로 전환해 "그림 그리는 개발자"를 텍스트가 아닌 사이트 경험 자체로 증명
