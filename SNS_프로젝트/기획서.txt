모바일 서비스 UI 기획서
=====================================

1. 서비스 기본 정보
-------------------------------------
서비스 이름: Once Upon A Time
서비스 주제: 동화책 홍보 및 리뷰 서비스
핵심 가치: 이미 출간된 유명 동화책 뿐만 아니라 새로 출간된 동화책의 내용 및 리뷰를 알아볼 수 있다.
타겟 사용자: 20~30대 여성

2. 디자인 컨셉
-------------------------------------
메인 색상: 노란색 (#FFD54F)
보조 색상: 하늘색 (#87CEEB)
전체 분위기: 밝고 귀여운 분위기
참고 사이트: sns레퍼런스2.jpg, sns레퍼런스1.jpg

3. 필요한 페이지 목록
-------------------------------------
[ ] 1. 로그인/회원가입 페이지
    - 기능: 이메일 로그인, SNS 로그인(카카오/구글), 비밀번호 찾기/재설정

[ ] 2. 메인 페이지 (홈 화면)
    - 기능: 탐색 탭에 연령별 동화책 분류(0~3세 / 4~7세 / 8~12세), 신간 동화책 홍보 배너 3개가 2초마다 변경되어 보여짐, 메인 페이지 상단 중앙에 검색 창, 메인 페이지 상단 좌측에 사이트 로고 배치
    - 특징: 카테고리가 한 줄씩 작성, 홍보 배너는 그림책 사진과 짧은 책 소개 및 제목으로 간결하고 명확하게 표현

[ ] 3. 검색 결과 페이지
    - 기능: 검색창에서 입력한 키워드로 동화책 제목/저자 기준 검색, 결과 목록을 카드 형식으로 표시
    - 특징: 검색 결과 없을 시 안내 문구 표시

[ ] 4. 게시물 목록 페이지
    - 기능: 최신 글 순으로 정렬이 기본, 최근 가장 많은 조회수 및 추천을 받은 게시글이 가장 상단에 강조되어 표시
    - 특징: '최신 글 순서대로, 오래된 순서대로, 추천 수 많은 순서대로' 게시글 순서 변경 칸이 우측 상단에 위치

[ ] 5. 게시물 상세 페이지
    - 기능: 동화책 표지 사진, 동화책 제목, 저자 이름, 출판사 안내, 별점, 한 줄 평, 읽은 후 소감 내용이 한 눈에 들어오게 정리
    - 특징: 게시글을 본 다른 사용자가 추천 및 댓글을 달 수 있는 칸, 부적절한 게시글/댓글 신고 기능

[ ] 6. 게시물 작성 페이지
    - 기능: 동화책 제목/저자 검색 후 기존 등록된 도서 선택(중복 등록 방지), 검색 결과가 없을 경우 신규 도서 정보(표지 사진, 제목, 저자, 출판사) 직접 입력, 별점, 한 줄 평, 읽은 후 소감 내용 작성

[ ] 7. 알림 페이지
    - 기능: 내 게시글에 달린 댓글/추천 알림, 팔로우 알림(팔로우 기능 도입 시) 목록 표시
    - 특징: 읽지 않은 알림 표시, 알림 클릭 시 관련 게시물로 이동

[ ] 8. 마이페이지
    - 기능: 사용자 정보 수정, 내가 쓴 게시글 정리된 칸, 내가 쓴 댓글 정리된 칸, 추천 누른 게시글 정리된 칸 포함
    - 특징: 카드 형식으로 깔끔하게 정리

4. 핵심 기능 정의
-------------------------------------
필수 기능 (꼭 있어야 하는 기능)
1. 프로필 생성 및 관리
2. 동화책 리뷰 게시물(별점, 한 줄 평, 소감) 작성, 수정, 삭제
3. 리뷰 게시글 추천(좋아요), 댓글
4. 도서 검색 및 연령별/신간 분류 탐색

부가 기능 (있으면 좋은 기능) - 3차 개발 항목
1. 관심 작가/사용자 팔로우 시스템
2. 추천(좋아요) 누른 리뷰 모음(컬렉션)

5. 모바일 UI 특징
-------------------------------------
네비게이션:
    - 하단 탭
    - 중앙에 게시물 작성하기 위한 버튼
    - 구성
        [홈][탐색][+작성][알림][마이]

주요 버튼 위치:
    - FAB : 하단 탭 중앙
터치 영역 크기:
    - 터치 가능한 요소의 최소 크기: 44x44px
    - 버튼 간 간격: 8px
세로/가로 모드: 세로 모드

6. 비기능 요구사항
-------------------------------------
지원 환경: 모바일 웹 우선 (반응형으로 태블릿/데스크톱 대응)
성능 기준: 페이지 전환 및 이미지 로딩 시 로딩 인디케이터 표시

7. 개발 우선순위
-------------------------------------
1차 개발: 로그인, 회원가입 페이지

2차 개발: 메인 페이지, 검색 결과 페이지, 게시물 목록/상세/작성 페이지

3차 개발: 알림 페이지, 마이페이지, 팔로우 시스템, 좋아요 컬렉션
